## 🛠️ المتطلبات المسبقة

### تثبيت أدوات PostgreSQL

#### Windows:
1. تحميل PostgreSQL من: https://www.postgresql.org/download/windows/
2. تشغيل المثبت واختيار "Command Line Tools"
3. إضافة مجلد bin إلى متغير PATH

#### macOS:
```bash
# باستخدام Homebrew
brew install postgresql

# أو باستخدام MacPorts
sudo port install postgresql15
```

#### Ubuntu/Debian:
```bash
sudo apt update
sudo apt install postgresql-client
```

#### CentOS/RHEL/Fedora:
```bash
# CentOS/RHEL
sudo yum install postgresql

# Fedora
sudo dnf install postgresql
```

## 🚀 تشغيل السكريبت

### على Windows:
1. فتح Command Prompt أو PowerShell
2. الانتقال إلى مجلد المشروع
3. تشغيل السكريبت:
```cmd
migrate-postgres-to-postgres.bat
```

### على Linux/macOS:
1. فتح Terminal
2. الانتقال إلى مجلد المشروع
3. إعطاء صلاحية التنفيذ:
```bash
chmod +x migrate-postgres-to-postgres.sh
```
4. تشغيل السكريبت:
```bash
./migrate-postgres-to-postgres.sh
```

## ⚠️ ملاحظات مهمة

### الأمان:
- **لا تشارك روابط قاعدة البيانات** مع أطراف ثالثة
- احتفظ بنسخة احتياطية قبل بدء العملية
- تأكد من صحة الروابط قبل التشغيل

### الشبكة:
- تأكد من اتصال إنترنت مستقر
- قد تحتاج VPN إذا كانت قاعدة البيانات محمية
- تحقق من إعدادات Firewall

### الوقت:
- العملية قد تستغرق عدة دقائق حسب حجم البيانات
- لا تقطع العملية أثناء التنفيذ

## 🔧 استكشاف الأخطاء

### خطأ في الاتصال:
```
ERROR: connection failed
```
**الحلول:**
- تحقق من صحة رابط قاعدة البيانات
- تأكد من إعدادات الشبكة
- جرب VPN إذا كانت المشكلة في الموقع الجغرافي

### خطأ في الصلاحيات:
```
ERROR: permission denied
```
**الحلول:**
- تحقق من صحة اسم المستخدم وكلمة المرور
- تأكد من صلاحيات المستخدم في قاعدة البيانات

### خطأ في أدوات PostgreSQL:
```
'pg_dump' is not recognized
```
**الحلول:**
- تثبيت PostgreSQL Client Tools
- إضافة مجلد bin إلى متغير PATH
- إعادة تشغيل Terminal/Command Prompt
